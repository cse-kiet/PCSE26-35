# 10 — End-to-End Pipeline

## Full pipeline from scratch

```bash
# 1. Install dependencies (once)
sudo apt install sumo sumo-tools
pip install matplotlib --break-system-packages

# 2. Clone / navigate to project
cd /path/to/PCSE26-35

# 3. Generate SUMO traces
make sumo

# 4. (Optional) Visualise vehicle movement
make visualise        # saves PNGs to src/sumo/
make sumo-gui         # opens live GUI (needs desktop)

# 5. Run MATLAB analysis
# Open MATLAB, then:
>> intersectionmultipolicy_sumo    % single-run comparison
>> run_trials                      % 30-trial statistics
>> stress_test                     % density + load sweeps
```

Total time: ~2 minutes for SUMO + traces, ~15 minutes for all MATLAB runs.

## All `make` targets

```
make                         same as make sumo
make sumo                    generate traces, skip steps already done
make sumo-rebuild            force regenerate network, routes, and traces
make sumo-rebuild SEED=N     rebuild with a specific random seed
make sumo-rebuild PERIOD=N   rebuild with a different vehicle density
make sumo-gui                open SUMO GUI for live playback
make visualise               generate trajectory, heatmap, speed PNG plots
make matlab-compare          run intersectionmultipolicy_sumo in batch mode
make matlab-trials           run run_trials in batch mode
make matlab-stress           run stress_test in batch mode
make clean                   delete all generated SUMO files
make help                    print all targets with descriptions
```

```
make sumo
  └── generate_traces.py
        ├── netconvert           builds intersection.net.xml
        ├── randomTrips.py       builds intersection.rou.xml
        └── sumo (headless)

        └── reads fcd_traces.csv
              ├── trajectory_plot.png
              ├── heatmap.png
              └── speed_profile.png

intersectionmultipolicy_sumo (MATLAB)
  └── addpath('src')
  └── parse_fcd('src/sumo/fcd_traces.csv')
  └── run_sumo_sim(policy, vehicles, ...)
        ├── get_vehicle_state
        ├── build_v2v_matrix → channel_model
        ├── generate_task_sumo

  └── same as above, repeated 30 times with rng(seed)

stress_test (MATLAB)
  └── same as above, in nested loops over vehicle_counts and arrival_rates
```

## Common workflows

### "I want to try a different traffic scenario"

```bash
make sumo-rebuild SEED=99 PERIOD=3   # dense traffic, different seed
make visualise                        # check the trajectories look reasonable
# then run MATLAB as normal
```

### "I want to test a more realistic channel model"

1. Open `src/channel_model.m`
2. Change `n = 2.7` to `n = 3.0` (denser urban)
3. Change `sigma = 4` to `sigma = 8` (heavier shadowing)
4. Re-run MATLAB — no need to regenerate SUMO traces

### "I want to compare two bandwidth settings"

```matlab
% In intersectionmultipolicy_sumo.m, change BANDWIDTH and run twice:
BANDWIDTH = 50;   run → note rates
BANDWIDTH = 10;   run → note rates
% compare manually or capture into a variable
```

### "I want to reproduce the exact paper results"

```bash
make sumo-rebuild SEED=42   # or just make sumo if already generated
```

```matlab
rng(42);
intersectionmultipolicy_sumo
```

For `run_trials`, no extra steps needed — it sets its own seed per trial.

### "I want to add a sixth policy"

1. Create `src/mypolicy.m` with signature:
   ```matlab
   function [doOff, target] = mypolicy(task, src, cands, vehicles, queues, BANDWIDTH)
   ```
2. Add it to the policies array in all three main scripts:
   ```matlab
   policies = {@greedy_policy, ..., @mypolicy};
   names    = {'Greedy', ..., 'MyPolicy'};
   ```
3. Run normally — it will appear in all tables and charts automatically.

## Troubleshooting quick reference

| Symptom | Cause | Fix |
|---|---|---|
| `randomTrips.py: No such file` | `SUMO_HOME` is wrong | `make sumo SUMO_HOME=/usr/share/sumo` |
| `netconvert: command not found` | SUMO not installed | `sudo apt install sumo sumo-tools` |
| `fcd_traces.csv: No such file` | `make sumo` not run yet | Run `make sumo` first |
| `sumo-gui: cannot connect to display` | No desktop / display | `export DISPLAY=:0` or use `make visualise` |
| `Undefined function parse_fcd` | MATLAB path not set | Run from project root, `addpath('src')` is in the script |
| MATLAB results differ between runs | No fixed seed | Add `rng(42)` at top of script |
| Fewer than 10 vehicles in traces | Period too long | `make sumo-rebuild PERIOD=3` |
